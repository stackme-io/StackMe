Architecture Summary

Data processing location Browser only   All analysis runs inside the user's browser via DuckDB-Wasm (WebAssembly SQL engine). No backend API for data.
File upload to server Never. Files are read by the browser File API and passed directly to the WASM engine in memory. No file is transmitted.
Column names / schema Never transmitted. Schema is inferred locally inside DuckDB-Wasm.
Query results / row data Never transmitted. Results stay in browser memory until the tab is closed.
Data retention Zero. No data is persisted. Closing the tab wipes everything.
Backend API for data None. The api/client.ts module exists for account features only (unrelated to data tools).

Network Requests

Vercel CDN (static assets) Page load only   JS, CSS, WASM binaries served from Vercel Edge. Standard CDN delivery, no user data.
Clerk (auth sessions) Session check only   Clerk handles authentication for account features. Makes a session-validity request on page load. Does not receive any file data, column names, or query results.
Analytics / tracking None. Vercel Analytics was removed. No page-view tracking, no cookies, no fingerprinting on data tool pages.
Third-party scripts None. No Google Analytics, Meta Pixel, Hotjar, or similar SDKs.
Requests after file upload Zero. Once a file is loaded, all processing is local. No outbound requests carry user data.

Hosting & Infrastructure

Static hosting Vercel Edge Network (SOC 2 Type II certified). Serves only pre-built static files.
Data center region Vercel Edge — closest point of presence. EU users served from EU edge nodes.
Server-side code None for data tools. Pure static SPA.
HTTPS Enforced. All traffic over TLS 1.2+.
Source code Open source, MIT License. Auditable at github.com/stackme-io/StackMe.

GDPR / Compliance

Personal data processed server-side None. File contents never reach a server.
Data Processing Agreement (DPA) Not required for core data tools — no personal data is processed by StackMe. May apply to Clerk (auth) if your organisation requires DPAs with all sub-processors.
Cookies Clerk sets a session cookie for authenticated users only. No tracking cookies.
Data subject rights (GDPR Art. 17) N/A — we hold no user data to delete.
Sub-processors Vercel (hosting), Clerk (authentication). Neither receives file data.
Independent verification (recommended)
Any reviewer can verify the zero-data-egress claim without trusting this document:

1. Open stackme-app.vercel.app/analyze-me in a browser
2. Press F12 → DevTools → Network tab → clear the log
3. Drop any CSV or JSON file onto the upload zone
4. Observe: zero outbound requests containing file contents, column names, or data values
